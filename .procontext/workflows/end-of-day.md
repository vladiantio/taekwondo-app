# End of Day Workflow — Taekwondo App

Checklist para consolidar trabajo al final de cada sesión de desarrollo.

**Tiempo estimado**: 5-10 minutos  
**Frecuencia**: Cada sesión de desarrollo (1-2x por día)

---

## ✅ Checklist EOD

### 1. Review Work Done (2 min)

- [ ] ¿Qué features/bugs completaste hoy?
- [ ] ¿Qué tests agregaste/actualizaste?
- [ ] ¿Qué documentación modificaste?

**Output**: Lista de items completados (para `session.md` update)

---

### 2. Update Session State (3 min)

Actualiza `.procontext/session.md`:

- [ ] Marca TODOs completados: `- [x]`
- [ ] Agrega nuevos TODOs descubiertos: `- [ ]`
- [ ] Documenta blockers encontrados: `🚧 Blockers`
- [ ] Agrega notas importantes: `💡 Notas de la Sesión`

---

### 3. Verify Code Quality (2 min)

```bash
# Lint check
npm run lint

# Format check
npm run format:check

# Type check
npx tsc --noEmit

# (Opcional) Run tests si modificaste lógica
npx playwright test
```

**Si hay errores**: Corregir antes de commit.

---

### 4. Git Commit (2 min)

```bash
# Stage changes
git add .

# Commit con mensaje convencional
git commit -m "feat(exams): add auto-scroll to current exam

- Implemented scroll-into-view for active exam
- Updated Exams.tsx component
- Added smooth scroll behavior

Refs: #47"

# (Opcional) Push si estás en feature branch
git push origin HEAD
```

**Convenciones**:
- `feat:` — Nueva feature
- `fix:` — Bug fix
- `docs:` — Documentación
- `style:` — Formatting, CSS
- `refactor:` — Refactor sin cambio de lógica
- `test:` — Tests agregados/modificados
- `chore:` — Build, deps, tooling

---

### 5. Document Decisions (Si aplica)

**Si tomaste decisión importante hoy**:

- [ ] Crear archivo en `.procontext/decisions/YYYY-MM-DD-decision-name.md`
- [ ] O promover a ADR en `.procontext/planning/adrs/` si es arquitectónica

**Ejemplo de decisión rápida**:
```markdown
# 2026-04-23: Usar Zustand persist para user preferences

**Context**: User settings (theme, language) se perdían en refresh.

**Decision**: Usar `zustand/middleware` `persist` con localStorage.

**Rationale**: Simple, 0 deps adicionales, sync automático.

**Alternatives**: Redux persist (overkill), manual localStorage (boilerplate).
```

---

### 6. Update Knowledge Base (Si aplica)

**Si aprendiste algo del dominio Taekwondo hoy**:

- [ ] Actualizar `.procontext/knowledge/glossary.md` (si hay términos nuevos)
- [ ] Actualizar `.procontext/knowledge/architecture/` (si cambió estructura)

**Ejemplo**:
```markdown
# Glosario — Taekwondo

**Tul**: Forma o patrón de movimientos (similar a kata en karate).
**Dan**: Grado de cinturón negro (1er Dan, 2do Dan, etc.).
**Dobok**: Uniforme de Taekwondo.
```

### 7. (Opcional) Create Daily Log

- **Con**: +13KB bundle, overkill para data estática?

## 🚫 Qué NO Hacer

❌ NO commitear trabajo sin linter check  
❌ NO dejar TODOs sin documentar en `session.md`  
❌ NO saltar verificación de tests (si modificaste lógica crítica)  
❌ NO documentar decisiones triviales (Ej: "agregué espacio en línea 42")
